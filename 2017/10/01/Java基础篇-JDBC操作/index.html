JDBC是如何操作数据库的

  Java数据库连接（Java DataBase Connectivity，JDBC）用于在Java程序中实现数据库操作的功能，它提供了执行SQL语句、访问各种数据库的方法，并为各种不同的数据库提供统一的操作接口，java.sql包中包含了JDBC操作数据库的所有类。通过JDBC操作数据库一般有以下几个步骤：

  （1）加载JDBC驱动器并将其注册到DriverManager中。将数据库的JDBC驱动加载到classpath中，在基于JavaEE的开发过程中，要将对应数据的JDBC驱动复制到WEB-INF/lib下。这通过java.lang.Class类的静态方法forName(String className)实现。 示例：

//连接MySql数据库，用户名和密码都是root
String url = “jdbc:mysql://localhost:3306/test” ;
String username = “root” ;
String passwd = “root” ;
try{
Connection conn = DriverManager.getConnection(url , username , passwd ) ;
}catch(SQLException se){
System.out.println(“数据库连接失败！”);
se.printStackTrace() ;
}

create database example;

use example;

create table Emp (
id int primary key,
name varchar(20),
age int,

)

import java.sql.*;

public class Test {
public static void main(String[] args) throws Exception {
String username = “root”;
String passwd = “root”;
String url = “jdbc:mysql://localhost:3306/example?useUnicode=true&characterEncoding=gbk”;
String driver = “com.mysql.jdbc.Driver”;
Connection conn = null;
Statement stmt = null;
ResultSet res = null;
try {
Class.forName(driver);
conn = DriverManager.getConnection(url, username, passwd);
stmt = conn.createStatement();
stmt.execute(“insert into Emp values(1, ‘Jobs’, 25, ‘male’)”);
stmt.execute(“insert into Emp values(2, ‘Lisa’, 19, ‘female’)”);
res = stmt.executeQuery(“select * from Emp”);
while(res.next()) {
System.out.println(res.getInt(1) + “ “ + res.getString(2) + “ “ + res.getInt(3) + “ “ + res.getString(4));
}catch(SQLException e) {
e.printStackTrace();
}finally {
try {
if(res != null) res.close();
if(conn != null) conn.close();
if(stmt != null) stmt.close();
}catch(SQLException e) {
System.out.println(e.getMessage());
}
}
}
}
}
//程序运行结果为：1 Jobs 25 male 2 Lisa 19 female

  虽然Statement对象和PreparedStatement对象都能够完成相同的功能，但相比之下，PreparedStatement具有以下优点：
  （1）效率更高。在使用PreparedStatement对象执行SQL命令时，命令会被数据库进行编译和解析，并放到命令缓冲区。然后，每当执行同一个PreparedStatement对象时，由于缓冲区种可以发现预编译的命令，虽然它会被再解析一次，但不会被再次编译，是可以重复使用的，能够有效提高系统性能，因此，如果要执行插入、更新、删除操作，最好使用PreparedStatement。
  （2）代码可读性和可维护性好。
  （3）安全性更好。使用PreparedStatement能够预防SQL注入攻击，所谓SQL注入，指的是通过SQL命令插入到Web表单递交或输入域名或页面请求的查询字符串，最终达到欺骗服务器而执行恶意SQL命令的目的。注入只对SQL语句的编译过程有破坏作用，而执行阶段只是把输入串作为数据处理，不再需要对SQL语句进行解析，因此也就避免了类似select * from user where name = ‘aa’ and password = ‘bb’ or 1 = 1的SQL注入问题的发生。
